About TaxTool

We present a comprehensive taxonomy for healthcare applications targeted to older adults

The world population is aging and this motivates the development of different software solutions focused on the older adults' health. However, there is a lack of a organization of the information used by these software. To address this gap, we present a taxonomy to support the development of software applications dealing with the older adults' health.

This taxonomy is composed by 87 characteristics and 21 categories that organize the information that should be handled by this kind of application. The assessment of our taxonomy was performed with ICT experts and healthcare professionals. Taxtool assists in the use of taxonomy.

How to use TaxTool

Here you can find instruction on how to use the tool.

As mentioned before, our taxonomy aims to assist the development of healthcare applications for older adults. It can be accessed through the link: full taxonomy. However, to facilitate the taxonomy usage, we developed this tool, entitled Taxtool. In addition to systematizing access to our taxonomy, the tool allows you to export Java or Javascript models from the fields of interest selected by the developer. To do this, perform the following steps:

  1. Select the application type. For example, application to assist older adult care or selfcare application.
  2. Choose which group of information (category) is relevante to you. E.g. relation with the reference person or disease information. Each group contains a set of fields that were classified as Essential (E), Optional (O), and None (N).
  3. After choosing the groups, you must select the target programming language. Thus, the tool will create a model considering the fields of the selected groups. Fields marked as None will not be included, and fields classified as Optional will be included with an annotation indicating this condition.
